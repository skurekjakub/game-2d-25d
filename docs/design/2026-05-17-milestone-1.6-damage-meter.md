## 1. What this delivers

A persistent **live damage meter** in the HUD plus an **end-of-run damage
summary** panel. Tracks total damage and DPS per weapon, rendered as
sorted rows with a growing percentage-of-total bar. Live meter caps at
top 5 weapons; end-of-run panel shows all.

**The player can answer in real time:**
- "Is my Aura pulling its weight, or is the Blaster carrying me?"
- "Is the Spread upgrade I just picked actually translating to damage?"
- "How much DPS do I average — is that improving as I level up?"

**The player can answer at end of run:**
- "What killed the most stuff this run?"
- "How did this run compare to my last one?" (humans compare, no auto-history)

## 2. Scope decisions (locked)

| Decision | Choice |
|---|---|
| Live UI | Yes — compact mini-meter, bottom-right corner, top 5 weapons by damage |
| End-of-run UI | Yes — full sorted breakdown, all weapons, total + average DPS + run duration |
| Stats tracked | Per-weapon: total damage, DPS (alive-time-weighted). Run-wide: total damage, run duration, average DPS |
| Granularity | Just damage + time. **No** hits/misses, **no** max-single-hit (defer to M2.x if needed). YAGNI applies. |
| State location | Dedicated `RunStats` Resource on `Game.run_state`, cleared on `start_run` |
| Aggregator | `DamageAggregator` autoload — sole writer to `RunStats` damage fields |
| Live render | `DamageMeterHud` Control, child of HUD scene |
| End render | `DamageSummaryPanel` Control, surfaced on `EventBus.run_ended` |
| Row widget | `DamageMeterRow` Control — single reusable widget, instanced by both Hud + Panel |

## 3. Architectural shape — strict SoC

Per project memory `feedback_oop_dry_soc`, M1.6 introduces strict separation
between data, aggregation, and presentation. No layer crosses concerns.

```
        ┌──────────────────────────────────────────────────────────────────┐
        │                          DATA                                     │
        │                                                                   │
        │   RunState  (existing — bootstrap/game.gd)                        │
        │     stats: RunStats                                               │
        │                                                                   │
        │   RunStats  (new — combat/stats/run_stats.gd)                     │
        │     damage_by_weapon: Dictionary[StringName, float]               │
        │     alive_time_by_weapon: Dictionary[StringName, float]           │
        │     run_started_at: float                                         │
        │     run_ended_at: float = 0.0                                     │
        │     ─────────────────────────────────────                          │
        │     add_damage(weapon_id: StringName, amount: float) -> void      │
        │     tick_alive_time(owned_ids: Array[StringName], dt: float)      │
        │     mark_run_started(now: float) -> void                          │
        │     mark_run_ended(now: float) -> void                            │
        │     total_damage() -> float                                       │
        │     elapsed_seconds() -> float                                    │
        │     average_dps() -> float                                        │
        │     dps_for(weapon_id) -> float                                   │
        │     percent_of_total_for(weapon_id) -> float                      │
        │     sorted_top_n(n: int) -> Array[RunStatsRow]                    │
        │     sorted_all() -> Array[RunStatsRow]                            │
        │                                                                   │
        │   RunStatsRow  (new — small RefCounted DTO for sorted output)     │
        │     weapon_id, display_name, damage, dps, percent                 │
        └──────────────────────────────────────────────────────────────────┘
                              ▲                              ▲
                              │ writes                       │ reads
                              │                              │
        ┌─────────────────────┴────────────────────┐  ┌─────┴─────────────────────────┐
        │              AGGREGATOR                   │  │            VIEW                 │
        │                                           │  │                                 │
        │   DamageAggregator  (new autoload)        │  │   DamageMeterRow (Control)      │
        │     - subscribes EventBus.damage_dealt    │  │     name_label, value_label,    │
        │     - subscribes EventBus.run_started     │  │     percent_bar                 │
        │     - subscribes EventBus.run_ended       │  │     apply(row: RunStatsRow)     │
        │     - _process: tick alive_time          │  │                                 │
        │       per owned weapon                    │  │   DamageMeterHud (Control)      │
        │     - resolves source Node → weapon_id    │  │     - child of HUD              │
        │       via WeaponInstance.data.id          │  │     - polls RunStats every       │
        │     - never reads RunStats fields back   │  │       0.25s (cheap; max 5 rows) │
        │     - never renders anything              │  │     - instances 5x DamageMeterRow│
        │                                           │  │                                 │
        │   DamageSourceResolver  (helper, static)  │  │   DamageSummaryPanel (Control)  │
        │     - resolve(source: Node) ->            │  │     - hidden until run_ended    │
        │         StringName id                     │  │     - instances N DamageMeterRow│
        │     - DRY: aggregator + tests both use it │  │     - shows total, dps, duration│
        └──────────────────────────────────────────┘  └─────────────────────────────────┘
```

**Invariants:**
- `DamageAggregator` is the **only writer** to `RunStats.damage_by_weapon`
  and `alive_time_by_weapon`. Views never mutate.
- `RunStats` is the **only canonical aggregation logic** — `dps_for`,
  `percent_of_total_for`, `sorted_top_n` live nowhere else.
- `DamageMeterRow` is the **only renderer of a single "name / value /
  bar" row** — both Hud and SummaryPanel instance it.

## 4. Live mini-meter (HUD)

**Where:** new child of `ui/hud.tscn`, anchored bottom-right corner.

**Behavior:**
- Hidden until first damage event fires (don't show empty rows).
- Shows up to 5 weapon rows, sorted by total damage descending.
- Refreshes every 0.25s (4 Hz) — enough to feel live, cheap enough not to
  matter. Uses a `Timer` node, not `_process`.
- Each row shows `<weapon_name>  <damage>  <bar>` where bar length is
  `damage / max_damage_in_row_set`. Max-relative makes the leader full,
  the trail proportional.
- Compact: ~280px wide × ~24px per row × 5 rows = ~140px tall.

**Mockup (during a run, 4 owned weapons, top 5 cap unused):**

```
                                                ┌─ Damage ─────────────────┐
                                                │ Blaster   12.5k ████████ │
                                                │ Aura       9.8k ██████   │
                                                │ Spread     6.2k ████     │
                                                │ Orbital    4.1k ██       │
                                                └──────────────────────────┘
```

**No top-of-screen DPS readout in the mini-meter.** That belongs in the
end-of-run summary, not the mid-run distraction surface.

## 5. End-of-run summary panel

**Where:** new scene `ui/damage_summary_panel.tscn`, instanced by HUD or a
top-level UI orchestrator on `EventBus.run_ended(victory)`.

**Behavior:**
- Slides/fades in over the dimmed-modulate HUD (M1.4 ran_ended modulate
  already applies).
- Centered, ~480×420 modal-ish panel (not actually modal — death/victory
  doesn't pause the game in M1.5; the panel is just informational).
- Shows total damage, run duration, average DPS at top.
- Lists all owned weapons sorted by damage descending — full breakdown,
  one row per weapon (not capped at 5).
- Each row: `<weapon_name>  <damage>  (<pct>%)  <bar>` — bar normalized
  to `damage / total_damage` here (not max-relative — the percentage is
  the meaningful number at run-end).
- Bottom: instruction text "Restart in M2.0" (M1.6 has no restart yet).

## 6. DPS calculation — alive-time-weighted

**Naive DPS:** `damage / total_run_time`. Wrong for late-acquired weapons.
A Spread picked at t=300s in a 600s run dividing by 600 underrates it 2×.

**Correct DPS:** `damage / alive_time_for_that_weapon`. Spread alive 300s,
gets divided by 300.

**Implementation:**
- `DamageAggregator._process(delta)`: walks
  `WeaponHost.owned_weapon_ids()` (group lookup → first player → host →
  list) and increments `RunStats.alive_time_by_weapon[id] += delta`.
- `RunStats.dps_for(id)` returns
  `damage_by_weapon.get(id, 0.0) / max(alive_time_by_weapon.get(id, 0.0),
  0.0001)`.
- Edge case: a weapon owned for 0.0 seconds (just acquired this frame)
  has no DPS; return 0.0 instead of dividing by zero.

**Run-wide average DPS** is `total_damage / elapsed_seconds()` (run-time,
not weapon-time — represents the player's overall throughput).

**Why `_process` not `_physics_process`:** alive-time is wall-clock for
display, and `Game.run_state.is_over` already gates `Game._process`'s
`time_elapsed` increment — `DamageAggregator._process` checks the same
flag for symmetry.

## 7. Source → weapon id resolution

`EventBus.damage_dealt(source: Node, target: Node, amount: float)` fires
with `source` = `BasicProjectile` / `AuraWeapon` / `OrbitalBlade`. The
aggregator needs the weapon **id** (`&"blaster"` / `&"aura"` /
`&"orbital"`).

**DRY helper:** `DamageSourceResolver.resolve(source: Node) -> StringName`

```
match source:
  BasicProjectile: ???   — projectile doesn't carry weapon id today
  AuraWeapon:      source.instance.data.id  if source.instance != null
  OrbitalBlade:    walk up to OrbitalWeapon parent, read parent.instance.data.id
  null / unknown:  return &""
```

**The Blaster/Spread case is a hole.** `BasicProjectile` is created by
`WeaponHost._spawn_single` and has no reference back to the weapon. Two
options:

1. **Stamp `weapon_id: StringName` on `BasicProjectile` at spawn time.**
   Single field, set in `_spawn_single`. The projectile then re-emits
   the id when it hits.

2. **Aggregator falls back to "uncategorized" for projectiles.** Bad —
   makes Blaster+Spread invisible in the meter.

**Pick option 1.** It's the minimal change and the projectile already
holds `damage` — adding `weapon_id` is in scope.

This makes `damage_dealt` semantics: source is **always** a node that
either is the weapon or carries a `weapon_id` field. Resolver checks for
the field first, then falls back to walking parent chain for scene-owned
weapons. Captured in `DamageSourceResolver`.

## 9. EventBus surface — no new signals

The damage meter is **pure read-side**. It reuses:
- `EventBus.damage_dealt(source, target, amount)` — already fires from
  BasicProjectile + AuraWeapon (M1.5) + OrbitalBlade (M1.5). Coverage
  audit was Round 1 of the M1.5 close-out.
- `EventBus.run_started` — resets the stats.
- `EventBus.run_ended(victory)` — surfaces the end-of-run panel.

**No** new signals. The view layer reads `Game.run_state.stats` directly
(it's the autoload's exposed state and there's no reason for the read to
go through a signal hop).

## 10. Test strategy

Each layer is independently testable.

| Suite | Coverage |

| `tests/test_run_stats.gd` | `add_damage` / `tick_alive_time` / `total_damage` / `elapsed_seconds` / `dps_for` / `percent_of_total_for` / `sorted_top_n` / `sorted_all` — pure-logic, no scene |
| `tests/test_damage_source_resolver.gd` | Stub nodes with `weapon_id` field → returns id. AuraWeapon stub → reads instance.data.id. OrbitalBlade stub with parent OrbitalWeapon → reads parent.instance.data.id. Unknown node → `&""` |
| `tests/test_damage_aggregator.gd` | Emit `damage_dealt` with stub source → RunStats updated. Emit `run_started` → stats reset. `_process` increments alive_time for owned weapons only |
| `tests/test_damage_meter_row.gd` | `apply(row)` updates labels + bar value |
| `tests/test_damage_meter_hud.gd` | Add damage to RunStats → meter shows N rows after refresh tick; sorted desc |
| `tests/test_damage_summary_panel.gd` | After `run_ended` emit, panel visible + populated with sorted_all |

Expected total: ~25 new tests. M1.5 ended at 155/155; M1.6 should land
~180.

## 11. Smoke checklist (Task 10 territory)

1. Start run. Damage meter HIDDEN (no damage yet).
2. Blaster kills first enemy. Meter appears: 1 row, `Blaster`, ~10 damage.
3. Multiple weapons after a few level-ups. Meter shows top 5 sorted; bar
   widths reflect rank.
4. Press F5 → damage log floods. Cross-check meter numbers match the
   running console totals over a few seconds.
5. Picking `+25% Blaster Damage` mid-run: subsequent hits log 12.5
   (rounded) and the meter's Blaster row rises faster.
6. Pre-Aura DPS for Aura: 0. Post-Aura at t+30s: ~24 DPS (3 enemies in
   range * 4dmg * 2 ticks/sec).
7. Die. Damage summary panel appears, lists all weapons, totals match
   the in-game meter, run duration is correct.
8. Replay → run starts → previous-run stats cleared (no carryover).

## 12. Risk register

| Risk | Likelihood | Mitigation |
|---|---|---|
| `BasicProjectile.weapon_id` field forgotten on spawn → "uncategorized" damage | medium | Resolver falls back to `&""`; aggregator drops `&""`-keyed damage (don't pollute meter); test_weapon_host asserts every spawned projectile has non-empty weapon_id |
| `_process` in DamageAggregator runs after Game.end_run → alive_time keeps ticking | medium | Same `Game.run_state.is_over` early-out as Game._process |
| Live meter refresh at 4Hz feels stale | low | Tune refresh interval; if needed move to per-frame `_process` but cap to "redraw only if RunStats hash changed" |
| End-of-run panel competes with future game-over UI in M2.0 | low | Panel is informational only; M2.0 will own pause/restart and can layer over it or replace it |
| Bar width uses `max_damage_in_set` (live) vs `total_damage` (panel) — inconsistent | accepted | Intentional. Live wants rank-at-a-glance; panel wants share-of-total. Document in `DamageMeterRow.apply()` API. |
| Damage events fire faster than aggregator can keep up | low | Signals are synchronous in Godot; backpressure is impossible. Real risk is `_process` order between Game (timer) and DamageAggregator (alive_time) — autoload order in project.godot decides. Spec: DamageAggregator AFTER Game so `is_over` is set before tick. |

- [ ] Live damage meter visible in-game corner, hidden until first damage
- [ ] Shows top 5 weapons sorted by total damage desc
- [ ] Bar widths reflect rank (leader full-width, others proportional)
- [ ] Refreshes ~4 Hz without per-frame work in handlers
- [ ] End-of-run summary appears on `run_ended`, all weapons listed
- [ ] Per-weapon DPS uses alive-time, not run-time

- [ ] No new EventBus signals; aggregator subscribes to existing ones only
- [ ] No archaeology comments anywhere in the diff
- [ ] All concerns separated: data / aggregator / view in distinct files,
      no cross-mutation, single canonical aggregation API on RunStats
