# Milestone 1.2 — SpawnerDirector + SpawnSchedule + 60-second two-phase wave Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace M1.1's hardcoded one-enemy-with-respawn-loop with a data-driven spawn system. Arena owns a `SpawnerDirector` Node that reads a `SpawnSchedule` resource (array of `SpawnPhase`s), spawns enemies at a budget-per-second rate at random off-camera positions around the player, transitions between two phases (slow → fast at t=30s), caps simultaneous enemies, and emits `EventBus.wave_completed` at t=60s. Player still indestructible (player HP / contact damage is M1.3).

**Architecture:** Data templates as `.tres` (`SpawnPhase`, `SpawnSchedule`); runtime brain as `SpawnerDirector extends Node` (one per arena, configured via `@export var schedule: SpawnSchedule`). SpawnerDirector accumulates a fractional spawn budget each `_process(delta)`, finds the current phase by `Game.run_state.time_elapsed`, refuses to spawn when at the concurrency cap, picks an enemy from the phase's weighted table, computes a spawn position on a circle around the player (radius = camera-visible diagonal + margin), and parents the new enemy to Arena's existing `Enemies` container. On phase boundary, emits `EventBus.spawn_phase_changed(idx)`. After the last phase's `starts_at_seconds + duration`, the spawner stops and emits `EventBus.wave_completed` exactly once. The XpGem-drop-on-enemy-killed behavior in Arena from M1.1 stays; the single-enemy spawn-and-respawn logic in Arena is removed.

**Tech Stack:** Godot 4.6.2, GDScript (statically typed), GdUnit4 — same as M1.0/M1.1.

**Phase 1 roadmap position:** Milestone 1.2 of 8. M1.1 (first combat / kill loop) is shipped at tag `milestone-1.1`. After this, **M1.3 — Player HP + contact damage + run end on death** is the next `writing-plans` cycle.

## Locked design decisions (encoded — do NOT re-litigate)

From the user's pre-plan brainstorm:

- **Scope:** Two `SpawnPhase` resources (0-30s slow, 30-60s fast). Exercises phase-switching logic. Both phases use only `basic_walker` for M1.2; weighted enemy tables are present in the data shape but currently single-entry.
- **Wave end:** Spawner stops spawning at t=60s. Existing enemies keep walking. EventBus emits `wave_completed` once; a simple `print("Wave 1 complete")` listener in Arena confirms it. No `run_ended`, no scene change, no HUD (HUD is M1.5+).
- **Player damage:** Deferred to M1.3. Enemies walk through player harmlessly in M1.2. No `ContactDamageComponent`, no player HP changes.
- **Spawn position:** Random point on a circle around the player at radius = `viewport_diagonal * 0.5 + spawn_margin` (camera-aware off-screen ring). Matches the design doc.
- **Concurrency cap:** 30 simultaneous enemies. When at cap, spawner accumulates budget without spawning so it bursts when an enemy dies.
- **No object pool yet:** `instantiate()` / `queue_free()` per enemy. Design doc says "decide on first perf measurement; start with `queue_free`." 30-enemy cap × 60s wave = max ~50 spawn events; not worth pooling.

## Project conventions (already proven in M1.0 + M1.1)

These were learned by iteration; do NOT re-litigate them. See `MEMORY.md` for the full memory index.

- **Test runner:** `./tests/run.sh [path]`. Path defaults to `tests/`; pass a single file like `tests/run.sh tests/test_spawn_schedule.gd` to scope.
- **Test pattern for Nodes:** `var thing: SomeType = auto_free(load("res://path.gd").new())` — `auto_free` from `GdUnitTestSuite` prevents leak warnings. For scenes: `var node = auto_free(load("res://path.tscn").instantiate())`.
- **Re-import after adding `class_name`:** `godot --headless --import --path .` once; the runner already does this on first run after a fresh checkout.
- **Headless smoke:** `godot --headless --quit --path .` — must exit 0, print `[Debug] enabled (debug build)` + `game-2d-25d boot ok — loading arena`, no `ERROR:` / `SCRIPT ERROR:` lines.
- **Static typing everywhere:** `var x: int`, `func foo(bar: String) -> void:`. Inferred via `:=` only when RHS makes the type obvious.
- **File order (GDScript style guide):** `@tool` / `class_name` / `extends` / signals / enums / constants / `@export` vars / public vars / private (`_`) vars / `_init` / `_ready` / public methods / private methods / inner classes (LAST).
- **Signal naming:** past tense (`damaged`, `died`, `enemy_killed`, `spawn_phase_changed`, `wave_completed`); never imperative.
- **Naming:** snake_case for files/vars/funcs, PascalCase for `class_name`/node types, CONSTANT_CASE for constants/enum members.
- **Resources are shared on load.** `load("foo.tres")` returns the same instance every time. Templates (`SpawnPhase`, `SpawnSchedule`) must be treated as immutable; runtime state lives elsewhere (in SpawnerDirector's instance fields).
- **Hand-written `.tres`/`.tscn` files in this project intentionally OMIT `uid="uid://..."` attributes.** Editor normalizes on first open. Do NOT add UID attributes manually. (See memory `godot-uid-handwritten-tres`.)
- **`@export` enum reorder gotcha:** any new enum members must be APPENDED, never inserted in the middle, OR explicitly assigned integer values. (See memory `godot-enum-export-reorder`.) Not applicable in M1.2 (no new enums) but worth remembering.
- **Adding physics nodes from inside `body_entered` chains:** must use `call_deferred("add_child", node)`. (See memory `godot-signal-callback-addchild`.) M1.2 spawns enemies from `_process`, not from a physics callback, so direct `add_child` is fine. The XpGem-drop from M1.1 already uses `call_deferred`.
- **Conventional commit per task:** `feat(world): ...`, `chore: ...`, `fix(...): ...`.
- **`.gdignore` reflex:** if any new addon ships a `test/` or `example/` subfolder with `class_name` declarations, drop a `.gdignore` in that subfolder immediately.

```gdscript
class_name SpawnSchedule
extends Resource

@export var phases: Array[SpawnPhase] = []


func total_duration_seconds() -> int:
	if phases.is_empty():
		return 0
	return phases[phases.size() - 1].end_time_seconds()


func phase_at_time(time_elapsed: float) -> SpawnPhase:
	for phase in phases:
		if time_elapsed >= float(phase.starts_at_seconds) and time_elapsed < float(phase.end_time_seconds()):
			return phase
	return null


func phase_index_at_time(time_elapsed: float) -> int:
	for i in phases.size():
		var phase: SpawnPhase = phases[i]
		if time_elapsed >= float(phase.starts_at_seconds) and time_elapsed < float(phase.end_time_seconds()):
			return i
	return -1
```

Create `world/spawner_director.gd`:

```gdscript
class_name SpawnerDirector
extends Node

# Local signals (also re-emitted to EventBus by the production paths;
# tests exercise these locally to stay pure-logic).
signal phase_advanced(phase_idx: int)
signal wave_ended

const ENEMY_SCENE := preload("res://combat/enemies/enemy.tscn")
const SPAWN_MARGIN: float = 64.0   # extra distance beyond the camera viewport diagonal

@export var schedule: SpawnSchedule
@export var enemies_container: Node2D
@export var max_concurrent_enemies: int = 30

var spawn_budget: float = 0.0
var _last_phase_index: int = -1
var _wave_ended_emitted: bool = false


func _ready() -> void:
	_last_phase_index = -1
	_wave_ended_emitted = false


func _process(delta: float) -> void:
	if schedule == null:
		return
	var time_elapsed: float = Game.run_state.time_elapsed
	_on_time_advanced(time_elapsed)
	var phase: SpawnPhase = schedule.phase_at_time(time_elapsed)
	if phase == null:
		return
	accumulate_budget(delta, phase.spawn_rate_per_sec)
	while spawn_budget >= 1.0:
		if _enemy_count_at_cap():
			break
		_try_spawn_one(phase)
		consume_one_spawn()


# Pure math: spawn position on a circle around player.
static func compute_spawn_position(player_pos: Vector2, radius: float, angle_rad: float) -> Vector2:
	return player_pos + Vector2(cos(angle_rad), sin(angle_rad)) * radius


func accumulate_budget(delta: float, rate_per_sec: float) -> void:
	spawn_budget += delta * rate_per_sec


func consume_one_spawn() -> void:
	spawn_budget = max(0.0, spawn_budget - 1.0)


# Track phase index transitions (local signal + EventBus re-emit).
func _on_time_advanced(time_elapsed: float) -> void:
	if schedule == null:
		return
	var idx: int = schedule.phase_index_at_time(time_elapsed)
	if idx != _last_phase_index and idx >= 0:
		_last_phase_index = idx
		phase_advanced.emit(idx)
		if is_inside_tree():
			EventBus.spawn_phase_changed.emit(idx)
	if idx < 0 and not _wave_ended_emitted and time_elapsed >= float(schedule.total_duration_seconds()):
		_wave_ended_emitted = true
		wave_ended.emit()
		if is_inside_tree():
			EventBus.wave_completed.emit()


func _enemy_count_at_cap() -> bool:
	if enemies_container == null:
		return false
	return enemies_container.get_child_count() >= max_concurrent_enemies


func _try_spawn_one(phase: SpawnPhase) -> void:
	if phase.enemy_data == null or enemies_container == null:
		return
	var player_pos: Vector2 = _player_position()
	var radius: float = _spawn_radius()
	var angle: float = randf() * TAU
	var spawn_pos: Vector2 = compute_spawn_position(player_pos, radius, angle)
	var enemy: Enemy = ENEMY_SCENE.instantiate()
	enemy.data = phase.enemy_data
	enemy.global_position = spawn_pos
	enemies_container.add_child(enemy)


# Player-position lookup via group; null-safe.
func _player_position() -> Vector2:
	var players := get_tree().get_nodes_in_group("player")
	if players.is_empty():
		return Vector2.ZERO
	var p: Node = players[0]
	if p is Node2D:
		return (p as Node2D).global_position
	return Vector2.ZERO


# Spawn radius = viewport diagonal / 2 + SPAWN_MARGIN.
# Resolved from the current viewport rather than a Camera2D node, because the
# camera lives under Player and we don't want SpawnerDirector to know about it.
func _spawn_radius() -> float:
	var vp := get_viewport()
	if vp == null:
		return 800.0   # safe headless fallback
	var rect: Rect2 = vp.get_visible_rect()
	# Account for zoom: zoomed-in (zoom > 1) means smaller visible world rect.
	var zoom: Vector2 = _camera_zoom()
	var world_w: float = rect.size.x / zoom.x
	var world_h: float = rect.size.y / zoom.y
	var diagonal: float = sqrt(world_w * world_w + world_h * world_h)
	return diagonal * 0.5 + SPAWN_MARGIN


func _camera_zoom() -> Vector2:
	var vp := get_viewport()
	if vp == null:
		return Vector2.ONE
	var cam := vp.get_camera_2d()
	if cam == null:
		return Vector2.ONE
	return cam.zoom
```

- The `spawn_budget` is intentionally NOT clamped at 1.0 — at high rates, we want it to accumulate past 1 so we burst-spawn after being capped.
