# Milestone 1.6 — Damage Meter Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> superpowers:subagent-driven-development to implement this plan
> task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a live mini damage meter + end-of-run damage summary,
backed by a dedicated `RunStats` resource and a `DamageAggregator`
autoload. Strict separation of data / aggregator / view per project
memory `feedback_oop_dry_soc`.

**Architecture:** Three-layer split. `RunStats` (data) owns counters
and read APIs. `DamageAggregator` (aggregator autoload) subscribes to
`EventBus.damage_dealt` + `run_started` + `run_ended`, writes to
`RunStats`, ticks alive-time per owned weapon every `_process`. View
layer (`DamageMeterRow`, `DamageMeterHud`, `DamageSummaryPanel`) reads
`RunStats` only — never mutates.

**Tech stack:** Godot 4.6.2-stable, GDScript (mandatory static typing),
GdUnit4. Approved design at
`docs/design/2026-05-17-milestone-1.6-damage-meter.md`.

- `godot_uid_handwritten_tres` — hand-written `.tres`/`.tscn` omit UIDs.

### EventBus.damage_dealt signal contract (load-bearing)

The signal carries `(source: Node, target: Node, amount: float)`. Three
emit sites exist or will exist:

- `combat/lookups/damageable.gd::try_damage` — weapon → enemy damage.
  source is the weapon node (BasicProjectile / AuraWeapon / OrbitalBlade).
- `player/player.gd::_on_damaged` — enemy → player contact damage.
  **source is `null`** (the player's HealthComponent doesn't know the attacker).
- (Future M1.7+) — TBD; honour the same contract.

DamageAggregator MUST handle the null-source case. `DamageSourceResolver.resolve(null)`
returns `&""`; `RunStats.add_damage(&"", amount)` drops via the empty-id
early-out. Net effect: **damage TAKEN by the player stays out of the
meter** (the meter tracks damage DEALT, per design §1).

Add the contract as a doc comment on the signal in `bootstrap/event_bus.gd`:

```gdscript
# Emitted on every damage application. source may be null (player contact-damage
# path); aggregator-side filters must drop null-source events. The damage meter
# (M1.6) tracks damage DEALT only.
signal damage_dealt(source: Node, target: Node, amount: float)
```

Add an explicit test in `tests/test_damage_aggregator.gd` (Task 6):

```gdscript
func test_player_contact_damage_does_not_pollute_meter() -> void:
    # Emulates Player._on_damaged emit shape (source=null, target=Player).
    var player_stub := Node.new()
    add_child(auto_free(player_stub))
    EventBus.damage_dealt.emit(null, player_stub, 25.0)
    await get_tree().process_frame
    assert_float(Game.run_state.stats.total_damage()).is_equal(0.0)
```

### Task 5 — use run_state.time_elapsed (drop _now_seconds)

**Override Task 5 Step 3 entirely.** No `_now_seconds()` helper, no
`Time.get_ticks_msec()`. Both `Game.start_run` and `Game.end_run` use
`run_state.time_elapsed`:

```gdscript
func start_run() -> void:
    run_state = RunState.new()
    run_state.stats.mark_run_started(0.0)  # time_elapsed origin
    _run_ended_emitted = false
    EventBus.run_started.emit()


func end_run(victory: bool) -> void:
    if _run_ended_emitted:
        return
    _run_ended_emitted = true
    run_state.is_over = true
    run_state.stats.mark_run_ended(run_state.time_elapsed)
    EventBus.run_ended.emit(victory)
```

Two parallel time sources (msec-since-boot vs `time_elapsed`) creates a
latent bug for M2.0 pause/restart — `time_elapsed` is gated by
`is_over` (ADR-0004) and matches the HUD timer the player sees.

Task 5 test fix: change `assert_float(stats.run_ended_at).is_greater(0.0)`
to `is_greater_equal(0.0)` — death in frame 0 would otherwise falsely
fail the assertion.

### Task 9 NumberFormat extraction

`DamageMeterRow._format_value` was Hud-private utility. Promote it to
its own module so DamageSummaryPanel doesn't reach into a Row widget for
formatting:

```gdscript
# ui/number_format.gd
class_name NumberFormat
extends RefCounted


# Compact integer-ish display: 1234 -> "1234", 12_500 -> "12.5k",
# 1_234_000 -> "1.2M".
static func compact(amount: float) -> String:
    if amount >= 1_000_000.0:
        return "%.1fM" % (amount / 1_000_000.0)
    if amount >= 10_000.0:
        return "%.1fk" % (amount / 1000.0)
    return "%d" % int(amount)
```

DamageMeterRow.apply() uses it. DamageSummaryPanel uses it. One source.

### DamageMeterRow.apply — no async await ready

Godot 4's `_ready` is end-of-frame deferred — `@onready var _name_label =
$NameLabel` is null when `apply()` is called synchronously after
`add_child(widget)`. Original plan worked around this with
`if not is_node_ready(): await ready`, which makes `apply` async + brittle.

**Cleaner fix:** drop the `@onready` vars. Use `$NameLabel` / `$ValueLabel`
/ `$PercentBar` directly inside `apply()`. The children exist in the
`.tscn` from instantiation, before `_ready` fires:

```gdscript
func apply(row: RunStatsRow) -> void:
    ($NameLabel as Label).text = row.display_name
    ($ValueLabel as Label).text = NumberFormat.compact(row.damage)
    ($PercentBar as ProgressBar).value = clamp(
        row.bar_fill_fraction * 100.0, 0.0, 100.0
    )
```

No `await`. Synchronous, safe.

```gdscript
func add_damage(weapon_id: StringName, amount: float) -> void:
    if weapon_id == &"" or amount <= 0.0:

    if dt <= 0.0:
        return
    for wid: StringName in owned_ids:
        alive_time_by_weapon[wid] = alive_time_by_weapon.get(wid, 0.0) + dt


func mark_run_started(now: float) -> void:
    run_started_at = now
    run_ended_at = 0.0


func mark_run_ended(now: float) -> void:
    run_ended_at = now


func elapsed_seconds() -> float:
    if run_ended_at > 0.0:
        return run_ended_at - run_started_at
    return 0.0


func elapsed_seconds_at(now: float) -> float:
    return now - run_started_at


func average_dps() -> float:
    var elapsed: float = elapsed_seconds()
    if elapsed <= 0.0:
        return 0.0
    return total_damage() / elapsed


func dps_for(weapon_id: StringName) -> float:
    var alive: float = alive_time_by_weapon.get(weapon_id, 0.0)
    if alive <= 0.0:
        return 0.0
    return damage_by_weapon.get(weapon_id, 0.0) / alive


func percent_of_total_for(weapon_id: StringName) -> float:
    var total: float = total_damage()
    if total <= 0.0:
        return 0.0
    return damage_by_weapon.get(weapon_id, 0.0) / total


func sorted_top_n(n: int, display_name_for: Callable) -> Array[RunStatsRow]:
    var rows: Array[RunStatsRow] = sorted_all(display_name_for)
    if rows.size() <= n:
        return rows
    return rows.slice(0, n)


func sorted_all(display_name_for: Callable) -> Array[RunStatsRow]:
    var rows: Array[RunStatsRow] = []
    for wid: StringName in damage_by_weapon.keys():
        var dmg: float = damage_by_weapon[wid]
        rows.append(
            RunStatsRow.new(
                wid, display_name_for.call(wid), dmg, dps_for(wid),
                percent_of_total_for(wid)
            )
        )
    rows.sort_custom(func(a: RunStatsRow, b: RunStatsRow) -> bool:
        return a.damage > b.damage
    )
    return rows
```

The `display_name_for: Callable` parameter keeps RunStats decoupled from
the WeaponRegistry — the aggregator (or test) supplies the lookup.

```gdscript
# bootstrap/damage_aggregator.gd
extends Node


func _ready() -> void:
    EventBus.damage_dealt.connect(_on_damage_dealt)


func _process(delta: float) -> void:
    if Game.run_state == null or Game.run_state.is_over:
        return
    var owned: Array[StringName] = _owned_weapon_ids()
    if owned.is_empty():
        return
    Game.run_state.stats.tick_alive_time(owned, delta)


func _on_damage_dealt(source: Node, _target: Node, amount: float) -> void:
    if Game.run_state == null:

    var weapon_id: StringName = DamageSourceResolver.resolve(source)
    Game.run_state.stats.add_damage(weapon_id, amount)


func _owned_weapon_ids() -> Array[StringName]:
    var tree := get_tree()
    if tree == null:

    var players: Array = tree.get_nodes_in_group("player")
    if players.is_empty():
        return []
    var host := players[0].get_node_or_null("WeaponHost") as WeaponHost
    if host == null:
        return []
    return host.owned_weapon_ids()
```

```gdscript
extends PanelContainer

const REFRESH_INTERVAL_SEC: float = 0.25
const MAX_ROWS: int = 5

@onready var _list: VBoxContainer = $List

func _ready() -> void:
    visible = false
    _timer = Timer.new()
    _timer.wait_time = REFRESH_INTERVAL_SEC
    _timer.one_shot = false
    _timer.timeout.connect(refresh_now)
    add_child(_timer)
    _timer.start()


func refresh_now() -> void:
    if Game.run_state == null:
        visible = false
        return
    var rows: Array[RunStatsRow] = Game.run_state.stats.sorted_top_n(
        MAX_ROWS, _display_name_for
    )
    if rows.is_empty():
        visible = false
        return
    visible = true
    _rebuild(rows)


func _rebuild(rows: Array[RunStatsRow]) -> void:
    for child: Node in _list.get_children():
        _list.remove_child(child)
        child.queue_free()
    var top_damage: float = rows[0].damage if not rows.is_empty() else 0.0
    for row_data: RunStatsRow in rows:
        var widget: DamageMeterRow = ROW_SCENE.instantiate()
        _list.add_child(widget)
        # Live meter uses MAX-relative bar (rank-at-a-glance) rather than
        # percent-of-total — overwrite the percent field with normalized rank.
        var rank_data := RunStatsRow.new(
            row_data.weapon_id, row_data.display_name, row_data.damage,
            row_data.dps,
            row_data.damage / top_damage if top_damage > 0.0 else 0.0
        )
        widget.apply(rank_data)


func _display_name_for(weapon_id: StringName) -> String:
    # Look up WeaponData on the player's host; fall back to id.
    var tree := get_tree()
    if tree != null:
        var players: Array = tree.get_nodes_in_group("player")
        if not players.is_empty():
            var host := players[0].get_node_or_null("WeaponHost") as WeaponHost
            if host != null:
                for w: WeaponInstance in host.weapons:
                    if w.data != null and w.data.id == weapon_id:
```

```gdscript
# ui/damage_summary_panel.gd
class_name DamageSummaryPanel
extends PanelContainer

const ROW_SCENE: PackedScene = preload("res://ui/damage_meter_row.tscn")

@onready var _list: VBoxContainer = $Layout/List
@onready var _total_label: Label = $Layout/Header/TotalLabel
@onready var _duration_label: Label = $Layout/Header/DurationLabel
@onready var _dps_label: Label = $Layout/Header/DpsLabel


func _ready() -> void:
    visible = false
    EventBus.run_ended.connect(_on_run_ended)


func _on_run_ended(_victory: bool) -> void:
    _populate()
    visible = true


func _populate() -> void:
    if Game.run_state == null:
        return
    var stats: RunStats = Game.run_state.stats
    _total_label.text = "TOTAL DAMAGE  %s" % DamageMeterRow._format_value(
        stats.total_damage()
    )
    _duration_label.text = "RUN DURATION  %s" % _format_mm_ss(
        stats.elapsed_seconds()
    )
    _dps_label.text = "AVERAGE DPS   %d" % int(stats.average_dps())
    for child: Node in _list.get_children():
        _list.remove_child(child)
        child.queue_free()
    var rows: Array[RunStatsRow] = stats.sorted_all(_display_name_for)
    for row_data: RunStatsRow in rows:
        var widget: DamageMeterRow = ROW_SCENE.instantiate()
        _list.add_child(widget)
        widget.apply(row_data)  # uses percent-of-total directly


func _display_name_for(weapon_id: StringName) -> String:
    var tree := get_tree()
    if tree != null:
        var players: Array = tree.get_nodes_in_group("player")
        if not players.is_empty():

            if host != null:
                for w: WeaponInstance in host.weapons:
                    if w.data != null and w.data.id == weapon_id:
                        return w.data.display_name
    return String(weapon_id).capitalize()


static func _format_mm_ss(seconds: float) -> String:
    var total: int = int(seconds)
```
